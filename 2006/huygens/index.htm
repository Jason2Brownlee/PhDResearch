Huygens Search & Optimization Competition (CEC'06)
Competition Software

Competition

As apart of the IEEE World Congress on Computational Intelligence conference 2006, specifically the Congress on Evolutionary Computation (CEC 2006), a number of competitions are held for computational intelligence algorithms (see the competition pages here and here). One competition that caught my interest was the Huygens Probe Competition in which algorithms are tested and compete on a number of black-box function optimisation problems. The clincher is that for each test problem the algorithm is exposed to only 1000 function evaluations are permitted, thus the problem of exploration and exploitation must be very carefully managed under this constraint of limited evaluations. For a complete description of the competition see here.

Software

The first thing I did for this competition was build a framework that would support a stack of standard off-line test problems and algorithms. The result was this function optimization software (14 algorithms, 30 problems, free source code, written in Java, etc.).

The second thing I did was write a test harness that would provide consistent benchmark results. The result is the free software provided on this page that permits any novice or expert computational intelligence practitioner to get involved in the competition without writing having to write a line of code.

The software uses the HTTP interface for the competition for function evaluations, and supports all of the algorithms supported by the afore mentioned function optimisation software as well as any additional third-party algorithms added to the framework. The software was written for speed, and supports batch-HTTP evaluations using a multithreaded evaluation engine (up to 50 simultaneous evaluations), which if improperly configured could cause harm to the competition evaluation server (so handle with care please). The software also supports proxy configuration and it should be noted is configured by default for the Swinburne University network proxy (non-Swinburne uses can simply change the properties file). A properties file is provided "huygens.properties" to configure all meaningful aspects of the client software. User login information for the competition is entered in to the software at runtime as a property of the problem. Graphs, and sample plots are provided to assist in tuning selected algorithms. Further, the software is written as a modular framework meaning that it is trivial to write alternative interfaces for batch runs or any other desired usage.

The software supports three problem modes as follows:

  1. Huygens Any Moon - Used for one-off algorithm tests. The user enters their login details and the moon they want to test against. The number of evaluations the can be reduced from 1000 for rapid proof of concept testing.
  2. Internal Benchmark - A benchmark interface is provided that captures test results across multiple moons and with multiple repetitions. A configured algorithm is evaluated on the first 10 moons (20_1 to 20_10). Benchmark results can be exported as csv (excel format) for offline algorithm comparison. This mode was invented for offline benchmarking using the single-moon training interface provided by the competition such that results would not be entered into the league table. It provides a means of comparing competitive algorithms "internally" before exposing results to the world via the server benchmark - very useful indeed for my own testing!
  3. Server Benchmark - This mode will test a configured algorithm on 10 unknown moons selected by the competition server, and results are entered in to the league table here. Progress bars are provided for user feedback of benchmark status.

Why release this software before the competition opens? Well I just like sharing, and I'm interested in third-party algorithm contributions or interesting algorithm configurations, so please contact me and tell me how you are using this software or framework? All comments and features requests can be directed to me.

Enjoy.
